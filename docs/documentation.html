Overview

Figma Importer for Unity is a two-part system that bridges design-to-development by automating the conversion of Figma UI designs into production-ready Unity UGUI prefabs.

The system consists of:

  • Figma Plugin - Exports frames as structured JSON with PNG images
  • Unity Package - Imports JSON and generates UGUI prefabs with full functionality

Data Flow

  1. Designer in Figma runs the plugin and exports a ZIP file (JSON + PNGs)
  2. Developer in Unity opens Tools > Figma > Import to Unity...
  3. FigmaNodeBuilder generates UGUI prefabs in Resources/Screens/
  4. Runtime uses UIScreenManager.Show("ScreenName") for navigation

Requirements

Unity

  • Unity 2021.3 or later
  • TextMeshPro (included with Unity)

Figma Plugin Development

  • Node.js and npm
  • Figma Desktop App

Installation

Unity Package

Copy the Assets/FigmaImporter folder into your Unity project's Assets folder. The package will be automatically recognized.

Figma Plugin

  1. Navigate to the FigmaPlugin folder
  2. Install dependencies and build: 
    cd FigmaPlugin
npm install
npm run build
  3. In Figma Desktop, go to Plugins > Development > Import plugin from manifest
  4. Select the FigmaPlugin/manifest.json file

Quick Start

1. Design in Figma

Create your UI design using frames. Name layers with prefixes to control import behavior:

  • BTN_Submit - Creates a button
  • IMG_Background - Creates an image
  • CTN_Header - Creates a container that processes children

2. Export from Figma

  1. Select a frame
  2. Run Plugins > Figma Importer for Unity
  3. Choose export scale and click Export Selected Frame
  4. Download the ZIP file

3. Import to Unity

  1. Extract the ZIP into your Unity project's Assets folder
  2. Open Tools > Figma > Import to Unity...
  3. Select the *_Screen.json file
  4. Click Import

4. Use at Runtime

// Navigate to a screen
UIScreenManager.Show("HomeScreen");

// Go back in history
UIScreenManager.Back();

// Instant navigation (no transition)
UIScreenManager.ShowImmediate("Settings");

Layer Naming Conventions

Use these prefixes in Figma layer names to control how elements are imported:

Prefix Behavior Unity Components Entry Animation
BTN_ Button - Baked as composite image Image, Button PopIn
IMG_ Image - Baked as PNG sprite Image FadeIn
ICON_ Icon - Baked as PNG sprite Image FadeIn
CTN_ Container - No image, processes children recursively RectTransform, LayoutGroup (if auto-layout) FadeSlideUp
9SLICE_ 9-Slice - Scalable background with borders Image (Sliced) None
TEXT Text - Native TextMeshPro (not layer name) TextMeshProUGUI None
Unprefixed Layers

Root children without prefixes are automatically baked as composite images for convenience.

Exporting from Figma

Export Modes

  • Export Selected Frame - Exports the currently selected frame
  • Export All Frames - Exports all frames in the page or section to one ZIP

Image Scale

Choose from 1x, 2x, 3x, or 4x resolution. This affects PNG quality only, not layout dimensions.

Export Contents

Each export creates:

  • {FrameName}_Screen.json - Layout data, hierarchy, constraints, text, interactions
  • {FrameName}_Images/ - PNG images organized by type

Component Variants

Figma component variants are fully supported for creating stateful UI elements like buttons.

Creating Button States

  1. Create a component set in Figma
  2. Add a "State" property with values: Default, Hover, Pressed, Disabled
  3. Design each state variant
  4. Export and import as normal
Auto-Wiring

When the importer detects Default/Hover/Pressed/Disabled state variants, it automatically adds FigmaButtonStateWirer to handle pointer events and state switching.

Runtime State Control

// Get the variant controller
var controller = GetComponent<FigmaVariantController>();

// Set state (exact match)
controller.SetState("State=Hover");

// Set state (partial match)
controller.SetState("Hover");

// Get available states
var states = controller.GetAvailableStates();

Prototype Interactions

Connect screens in Figma's prototype mode and navigation is automatically wired in Unity.

Supported Interactions

Figma Action Unity Behavior
Navigate to frame UIScreenManager.Show("FrameName")
Back UIScreenManager.Back()
Open URL Application.OpenURL(url)

The FigmaNavigationButton component is automatically added to elements with prototype interactions.

Importing to Unity

Import Window

Open Tools > Figma > Import to Unity... to access the importer.

JSON File Mode

  1. Click Browse to select the *_Screen.json file
  2. Configure import settings
  3. Click Import

Generated Output

  • Resources/Screens/{FrameName}.prefab - Main screen prefab
  • {ExportFolder}/Prefabs/ - Component prefabs
  • Imported sprites in the export folder

Import Settings

Setting Description Default
Pixels Per Unit Sprite scale factor (affects sprite sizing) 100
Remove Orphans Delete nodes that were removed in Figma On
Create Canvas Auto-create Canvas if missing On
Create EventSystem Auto-create EventSystem if missing On
Setup UIScreenManager Auto-create navigation manager On
Add Entry Animations Add FigmaEntryAnimation components On
Stagger Delay Delay between sibling animations 0.05s

Optional Prefabs

  • 3D Environment Prefab - Creates Resources/Environments/{FrameName}_Environment.prefab
  • Screen Transition - Creates Resources/Transitions/{FrameName}_Transition.prefab
  • Video Plane - Creates video playback quad with RenderTexture

Re-importing

When you re-import an updated design, the system intelligently preserves your customizations.

What's Preserved

  • onClick event handlers
  • Manually added MonoBehaviours
  • Custom component references
  • Inspector modifications to runtime components

What's Updated

  • Positions and sizes
  • Layout group settings
  • Images and sprites
  • Text content and styling
  • New nodes added in Figma

Orphan Cleanup

Nodes deleted in Figma are removed from Unity (when "Remove Orphans" is enabled). Each node is tracked by its unique Figma ID via the FigmaNodeKey component.

Auto Re-import

Enable file watching to automatically re-import when export files change.

Setup

  1. In the Import Window, expand Auto-Reimport Settings
  2. Click Add Folder and select your export directory
  3. The system watches for changes to *_Screen.json and *_Images/*.png
Debouncing

Changes are debounced with a 0.5 second delay to handle rapid file modifications.

Screen Navigation

The UIScreenManager singleton handles all screen navigation at runtime.

API Reference

// Navigate to a screen with transition
UIScreenManager.Show("HomeScreen");

// Navigate instantly (no transition)
UIScreenManager.ShowImmediate("Settings");

// Go back in navigation history
UIScreenManager.Back();

// Get the current screen name
string current = UIScreenManager.CurrentScreen;

// Check if a transition is in progress
bool transitioning = UIScreenManager.IsTransitioning;

Events

// Subscribe to navigation events
UIScreenManager.OnTransitionStart += (from, to) => {
    Debug.Log($"Navigating from {from} to {to}");
};

UIScreenManager.OnTransitionComplete += (screen) => {
    Debug.Log($"Now showing {screen}");
};

Screen Loading

Screens are loaded from Resources/Screens/{name}. The manager also auto-loads matching environment prefabs from Resources/Environments/{name}_Environment.

Screen Transitions

FadeTransition

Fades to a color (default black) and back when switching screens.

Properties

Property Description
outDuration Time to fade out current screen
holdDuration Time to hold while fully covered
inDuration Time to fade in new screen
fadeColor Color to fade to

CameraTransition

Applies camera effects during screen transitions.

Effect Types

  • Shake - Perlin noise-based camera shake with configurable intensity
  • ZoomPulse - FOV change with smooth sine curve
  • DollyForward - Camera moves forward then back
  • Drift - Smooth camera drift with position offset

Entry Animations

Entry animations play when screens are shown, creating polished UI transitions.

Animation Types

Type Description
FadeInAlpha animates from 0 to 1
PopInScale from 0 to 1 with spring overshoot
SlideFromLeftSlides in from the left
SlideFromRightSlides in from the right
SlideFromTopSlides in from the top
SlideFromBottomSlides in from the bottom
FadeSlideUpFades in while sliding up
FadeSlideDownFades in while sliding down
FadeSlideLeftFades in while sliding left
FadeSlideRightFades in while sliding right
FadePopInFades in with scale spring

Configuration

var anim = GetComponent<FigmaEntryAnimation>();

// Configure animation
anim.animationType = EntryAnimationType.PopIn;
anim.duration = 0.4f;
anim.delay = 0.1f;
anim.popOvershoot = 1.2f;

// Manual playback
anim.PlayAnimation();
anim.ResetToOriginal();

Staggered Animations

Sibling elements animate with a stagger delay, creating a cascading effect. The delay is calculated as siblingIndex * staggerDelay.

Variant Controller

The FigmaVariantController manages state switching for components with variants.

var controller = GetComponent<FigmaVariantController>();

// Set state with exact key
controller.SetState("State=Hover, Size=Large");

// Set state with partial match
controller.SetState("Hover"); // Matches "State=Hover"

// Get current state
string state = controller.GetCurrentState();

// List available states
string[] states = controller.GetAvailableStates();

Architecture

Export Format (v7)

The JSON export bundle contains:

  • version - Format version (currently 7)
  • documentName, frameName, frameId - Document metadata
  • width, height - Frame dimensions
  • nodes - Hierarchical node tree
  • components - Figma component definitions
  • componentSets - Variant set definitions

Troubleshooting

Images not appearing

  • Ensure the *_Images folder is in the same directory as the JSON file
  • Check that image file names match references in the JSON
  • Verify sprites are imported with correct settings (Texture Type: Sprite)

Text not rendering

  • Ensure TextMeshPro is installed (Window > TextMeshPro > Import TMP Essential Resources)
  • Check font assets are available in the project

Layout not matching Figma

  • Verify constraints are set correctly in Figma (not just visual positioning)
  • Check that auto-layout is enabled on the parent frame in Figma
  • Ensure Pixels Per Unit matches your design scale

Navigation not working

  • Verify UIScreenManager exists in the scene
  • Check that screen prefabs are in Resources/Screens/
  • Ensure prototype interactions are set up in Figma before export

Re-import losing customizations

  • Customizations on the prefab root may be overwritten
  • Add custom scripts to child objects for better preservation
  • Use prefab variants for extensive customizations
